爬虫基本结构

正常通过浏览器可以获取的数据，爬虫都可以获取,爬虫的本质是一个服务端，实现的功能类似于浏览器.是一种自动获取网页内容的程序。是搜索引擎的重要组成部分.我们需要干的事情其实就两件事情：找到我们需要的网页，然后把他们一个一个处理一遍。所以，爬虫的核心模块有三个：请求、解析、储存

  • 对于单个页面来说，入口的来源有两种，一种是已知地址,另外一种是你可以通过前面爬下来的网页中获取入口.例如浏览网页的过程，用户输入网址之后，经过 DNS 服务器，找到服务器主机，向服务器发出一个请求，服务器经过解析之后，发送给用户的浏览器 HTML、JS、CSS 等文件，浏览器解析出来，用户看到的网页实质是由 HTML 代码构成的，通过分析和过滤这些 HTML 代码，实现资源的获取。

http协议

所谓HTTP协议，中文名是超文本传输协议。HTTP是基于TCP/IP的关于数据如何在万维网中如何通信的协议,这是一种可靠的可以把各种各样格式的文件在互联网上传输的协议。HTTP协议的传输主要通过HTTP报文实现的.

  1. 客户端（比如浏览器） ：发送请求报文到服务器,客户端请求的方法也是不一样的,最常见的的一种是get方法，一种是post方法，两种的具体区别及联系参见文章请求
  2. 服务器：接收请求报文并处理
  3. 服务器：发送响应报文给客户端
  4. 客户端：处理接收到的内容。

URL的含义

URL，即统一资源定位符，也就是我们说的网址,互联网上的每个文件都有一个唯一的URL，它包含的信息指出文件的位置以及浏览器应该怎么处理它。

  • URL 的格式由三部分组成：
  1. 第一部分是协议(或称为服务方式)。
  2. 第二部分是存有该资源的主机IP地址(有时也包括端口号)。
  3. 第三部分是主机资源的具体地址，如目录和文件名等。
  • IP地址.在生活中我们使用具有上网功能的电子设备都有IP地址，就跟每个人都有自己的名字一样。IP地址分为IPV4 IPV6，我们所说的的IP地址指的是IPV4的地址。IP地址是由两部分组成的，网络部分和主机部分.

爬虫必备 requests

Requests是一个基于Apache2协议开源的Python HTTP库，号称是“为人类准备的HTTP库”,用于request.
